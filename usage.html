shift

WebAssembly runtime for Python, Lua, Ruby and etc.

Online demo: https://shift.js.org/

NPM Package: https://www.npmjs.com/package/shift-wasm

Usage

NPM Package

Install:

1npm install shift-wasm

Files:

LanguageWASM FileData File
Pythonshift-wasm/assets/python.wasmshift-wasm/assets/python.data
JavaScriptshift-wasm/assets/qjs.wasm
Luashift-wasm/assets/lua.wasm
Bashshift-wasm/assets/bash.wasm
Rubyshift-wasm/assets/ruby.wasm
Schemeshift-wasm/assets/chibi.wasmshift-wasm/assets/chibi.data
Cshift-wasm/assets/picoc.wasm

Examples in Vite and React environments:

1import { useEffect } from "react"
2import { makeConfig, python } from "shift-wasm"
3
4// get location of .wasm and .data files
5import wasm from "shift-wasm/assets/python.wasm?url";
6import data from "shift-wasm/assets/python.data?url";
7
8function App() {
9    useEffect(() => {
10        // the code to run
11        const code = "print('hello world')";
12
13        // callback to get result
14        const fn = (text:string) => (document.querySelector("#output") as HTMLElement).innerText = text;
15
16        // run the code
17        python(makeConfig(code, fn, wasm, data));
18    })
19    
20    return <pre id="output"></pre>
21}
22
23export default App

Different build tools use different ways to get location of .wasm and .data files.

Additionally, you may need to exclude ws, for example, vite and rollup:

1import { defineConfig } from 'vite'
2
3// https://vite.dev/config/
4export default defineConfig({
5  // ...
6  build: {
7    rollupOptions: {
8      external: ["ws"],
9    }
10  },
11})

ES6 external module

1<html>
2    <body>
3        <!-- Create a element to display output -->
4        <pre id="output"></pre>
5        <script type="module">
6            // Load module
7            import python from "https://shift.js.org/wasm/python.js";           // interpreter runtime
8            import makeConfig from "https://shift.js.org/wasm/common.js" ;      // interpreter config
9
10            // the code to run
11            const code = "print('hello world')";
12
13            // callback to get result
14            const fn = (text) => document.querySelector("#output").innerText = text;
15
16            // run the code
17            python(makeConfig(code, fn));
18        </script>
19    </body>
20</html>

makeConfig in external module is different from NPM package, it does not need the wasm and data parameters.

Refer page in <iframe>

1<html>
2    <body>
3        <!-- iframe to display shift -->
4        <iframe id="code" title="Shift" style="width: 100%; height: 600px; border: 0;"></iframe>
5
6        <script>
7            const lang = 'lua';                     // language
8            const input_text = '';                  // data of STDIN
9            const code_text = 'print("hello")';     // code to run
10
11            // generate URL
12            const input = encodeURIComponent(btoa(encodeURIComponent(input_text)));
13            const code = encodeURIComponent(btoa(encodeURIComponent(code_text)));
14            const url = `https://shift.js.org/#lang=${lang}&input=${input}&code=${code}`;
15            
16            // set iframe src
17            document.querySelector("#code").src = url;
18        </script>
19    </body>
20</html>
